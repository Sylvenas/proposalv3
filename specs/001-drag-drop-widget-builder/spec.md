# Feature Specification: Drag-and-Drop Widget Builder for Proposals

**Feature Branch**: `001-drag-drop-widget-builder`
**Created**: 2026-03-19
**Status**: Draft
**Input**: User description: "我想创建一个通过拖拽widget完成网页渲染的原型，假设用户要创建proposal，那么一般来说会包含几个基本的信息：公司信息（包含名称，logo）,销售信息（姓名，电话，邮箱等等），product list(产品名称，价格，数量，图片)，付款计划（总金额，分三次付款，每次多少钱）；相当于我们有四个独立的小模块，那么用户可以通过在拖拽的方式在"画布"上自由的移动这四个小模块的位置和大小来定制自己公司的proposal的展示形式；页面整体可以分为三列；最左侧为小组件列表；中间为画布，最右侧为预览"

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 拖拽组件到画布 (Priority: P1)

用户打开编辑器页面，在左侧组件面板中看到可用的 Proposal 组件（公司信息、销售信息、产品列表、付款计划），将任意组件拖拽到中间画布上，组件出现在放置位置并在画布内可见。

**Why this priority**: 这是整个功能的核心交互——没有拖拽放置能力，其余功能均无法成立。完成后即可得到一个可用的最小原型。

**Independent Test**: 打开页面，将"公司信息"组件从左侧面板拖入画布，验证组件在画布中正确渲染，即为独立可测试的 MVP。

**Acceptance Scenarios**:

1. **Given** 左侧面板显示所有可用组件，**When** 用户将"公司信息"组件拖到画布上，**Then** 该组件出现在鼠标释放的位置，并显示公司名称和 logo 占位内容。
2. **Given** 画布中已有一个组件，**When** 用户拖入第二个不同类型的组件，**Then** 两个组件均在画布上独立显示，互不覆盖。
3. **Given** 用户在左侧面板拖动组件，**When** 拖动过程中，**Then** 显示拖拽的视觉反馈（半透明预览或高亮放置区域）。

---

### User Story 2 - 在画布上移动和调整组件大小 (Priority: P2)

用户已将若干组件添加到画布后，可以在画布内自由拖动每个组件更换位置，也可以拖拽组件边角/边框来改变其尺寸，以满足不同排版需求。

**Why this priority**: 自由布局是本功能差异化的核心价值；仅能"放置"但无法重排，对实际使用意义有限。

**Independent Test**: 在画布中放置两个组件，拖动其中一个到不同位置后，验证位置已更新且不影响另一个组件；再拖拽角落调整大小，验证尺寸改变。

**Acceptance Scenarios**:

1. **Given** 画布中存在一个组件，**When** 用户在组件上按住并拖动，**Then** 组件跟随鼠标移动，释放后停留在新位置。
2. **Given** 画布中存在一个组件，**When** 用户拖拽组件的调整手柄（边角或边缘），**Then** 组件尺寸实时变化，释放后保持新尺寸。
3. **Given** 多个组件在画布中，**When** 移动某一组件，**Then** 其他组件位置不受影响。

---

### User Story 3 - 实时预览最终效果 (Priority: P3)

用户在编辑画布的同时，右侧预览区域同步展示 Proposal 的最终呈现效果，包括组件的实际内容、布局位置和尺寸，无需额外操作即可看到客户将看到的结果。

**Why this priority**: 预览是编辑器的重要辅助功能，让用户对布局效果形成直观判断，但不影响基础编辑能力。

**Independent Test**: 在画布中放置并移动组件，验证右侧预览区域实时反映最新布局与内容。

**Acceptance Scenarios**:

1. **Given** 用户将组件拖入画布，**When** 组件放置完成，**Then** 右侧预览区域立即显示该组件的实际内容和位置。
2. **Given** 预览区已显示当前布局，**When** 用户在画布中移动或调整任意组件，**Then** 预览区同步更新，无需手动刷新。
3. **Given** 查看预览区域，**When** 与画布对比，**Then** 预览内容与画布布局在相对位置和比例上保持一致。

---

### User Story 4 - 从画布移除组件 (Priority: P4)

用户可以将已添加到画布的组件删除，重新规划 Proposal 的结构。

**Why this priority**: 删除能力保证编辑流程的完整性，但优先级低于核心放置和移动功能。

**Independent Test**: 在画布中添加一个组件，通过删除操作将其移除，验证画布和预览区均不再显示该组件。

**Acceptance Scenarios**:

1. **Given** 画布中存在一个组件，**When** 用户点击该组件上的删除按钮，**Then** 该组件从画布和预览区消失。
2. **Given** 删除组件后，**When** 用户查看左侧面板，**Then** 被删除的组件类型仍可在面板中选用，支持重新添加。

---

### Edge Cases

- 当用户将组件拖拽到画布边界之外时，组件应自动吸附到最近的有效边界内，不允许超出画布范围。
- 当同一类型组件重复添加到画布时（如添加两个"公司信息"），系统按假设每类限制一个实例，但此行为待澄清（见 Assumptions）。
- 当画布中组件数量较多，整体高度超出可视区时，画布区域应出现滚动条，而非裁剪组件内容。
- 当用户快速连续拖动时，画布和预览区应保持状态一致，不出现错位或残影。

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在页面左侧呈现组件面板，列出所有可用的 Proposal 组件（公司信息、销售信息、产品列表、付款计划）。
- **FR-002**: 系统 MUST 支持将左侧面板中的组件通过拖拽方式放置到中间画布上。
- **FR-003**: 系统 MUST 在画布上为每个组件渲染对应的内容区域，展示该组件的字段占位信息（如公司名、logo、姓名、电话、产品名称/价格/数量/图片、付款计划明细等）。
- **FR-004**: 用户 MUST 能够在画布内拖动已放置的组件，改变其位置。
- **FR-005**: 用户 MUST 能够通过拖拽调整手柄改变画布中组件的宽高尺寸。
- **FR-006**: 系统 MUST 在右侧提供实时预览区域，同步展示画布当前布局的最终效果。
- **FR-007**: 用户 MUST 能够从画布中删除已放置的组件。
- **FR-008**: 系统 MUST 在拖拽过程中提供视觉反馈，包括拖拽预览和有效放置区域的高亮指示。
- **FR-009**: 系统 MUST 将页面布局为三列结构：左侧组件面板 / 中间画布 / 右侧预览。
- **FR-010**: 当组件位置超出画布边界时，系统 MUST 将其限制在画布有效区域内。

### Key Entities

- **Widget（组件定义）**: 代表一种可复用的 Proposal 内容模块，包含类型标识（如 `company-info`）、展示名称、图标及默认字段集合。
- **CanvasItem（画布实例）**: 组件在画布上的放置实例，包含唯一 ID、关联的 Widget 类型、在画布中的位置（水平/垂直坐标）及尺寸（宽度/高度）。
- **Canvas（画布）**: 中间编辑区域，持有所有 CanvasItem 的集合及画布自身的尺寸边界。
- **PreviewPane（预览面板）**: 基于 Canvas 当前状态实时渲染的只读输出视图，与画布布局保持同步。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可在 30 秒内完成"从面板拖拽一个组件到画布并看到预览"的完整操作，无需任何文档指引。
- **SC-002**: 画布上的组件移动和大小调整操作响应流畅，用户感知无明显卡顿。
- **SC-003**: 右侧预览区在每次画布变更后自动更新，用户无需手动触发刷新操作。
- **SC-004**: 所有四个预设 Proposal 组件（公司信息、销售信息、产品列表、付款计划）均可独立拖入画布并正确渲染对应字段内容。
- **SC-005**: 首次接触该工具的用户，在 5 分钟内能够独立构建出包含至少 2 个组件的 Proposal 布局。

---

## Assumptions

- **原型范围**: 本功能为交互原型，不需要持久化存储——页面刷新后布局数据可以丢失。
- **静态内容**: 组件内数据（如公司名称、销售人员姓名、产品信息）在原型阶段以静态占位内容展示，不需要用户填写真实数据。
- **自由定位布局**: 画布采用绝对坐标定位（而非网格/列约束布局），每个组件可放置在画布内任意位置。
- **每类组件单实例**: 每种组件类型默认只允许在画布中添加一次（如只能有一个"公司信息"模块）。如业务需要允许重复添加，需修订 FR-002。
- **预览等比缩放**: 预览区使用与画布相同的布局比例展示，不做额外美化处理。
- **三列固定布局**: 页面整体三列结构中，左侧组件面板和右侧预览列宽度固定，中间画布列宽度可弹性伸缩。
